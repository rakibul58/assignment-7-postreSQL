1. What is PostgreSQL?

Answer: PostgreSQL is an open source RDBMS( Relational Database Management System ) which was invented at the Berkeley Computer Science Department, University of California in 1986. It is know for it's robustness, scalability, and support for advanced features.

2. What is the purpose of a database schema in PostgreSQL?

Answer: A PostgreSQL database schema offers administrative and structural control for database objects, such as tables and views. It makes maintenance easier. It helps dealing with the database more easily.

3. Explain the primary key and foreign key concepts in PostgreSQL.

Answer: Primary key is the unique column or collection of columns in a table row. Primary key must not be null. Foreign key is the key that comes from another table which is usually the primary key of that other table and it references the row of that other table.

4. What is the difference between the VARCHAR and CHAR data types?

Answer: The main difference of VARCHAR and CHAR is that CHAR is fixed length while VARCHAR is supports variable length.

5. Explain the purpose of the WHERE clause in a SELECT statement.

Answer: WHERE clause specifies the row or rows the SELECT statement will be performed on.

6. What are the LIMIT and OFFSET clauses used for?

Answer: LIMIT is used for how much data one want to fetch while OFFSET is used for from which index the data should be fetched.

7. How can you perform data modification using UPDATE statements?

Answer:  We can use the following syntex to perform update statements:

UPDATE table_name
SET column1 = value1, column2 = value2, ...
WHERE condition;

8. What is the significance of the JOIN operation, and how does it work in PostgreSQL?

Answer: The JOIN operation enables us to perform query between multiple tables. There are different types for JOIN such LEFT JOIN, RIGHT JOIN, FULL JOIN, INNER JOIN etc.

9. Explain the GROUP BY clause and its role in aggregation operations.

Answer: GROUP BY allows to group the data by data in a column which helps us to know how many data is in a particular data like how many student whose age is 23. It also shows the all the data in that particular group.

10. How can you calculate aggregate functions like COUNT, SUM, and AVG in PostgreSQL?

Answer: The syntex for COUNT, SUM, and AVG in PostgreSQL are give bellow: 

COUNT: SELECT COUNT(*) AS total_students FROM students;
SUM: SELECT SUM(frontend_mark) AS total_frontend_marks FROM students;
AGE: SELECT AVG(age)::NUMERIC(10,2) AS average_age from students;
